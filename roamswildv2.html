RoamsWild

A trip planner for the places GPS forgets — real itineraries for road-based travel on US public lands, from a soft prompt like "4 days in Southern Utah, no 4WD." Built solo.

Visit RoamsWild
The Problem

Multiple sources, no agreement.

RoamsWild serves two travelers who are usually the same person — the overlander asking "can my rig get there, and is it legal to camp?" and the photographer asking "will the light be any good when I arrive?" Underneath both is one question: can I trust this enough to drive hours off-grid to it?

Crowdsourced noise, no curation

Apps bridge fragmented public-land data with user-submitted pins that are often outdated, vague, overcrowded, or on private land.

No legal land logic

Wild-camping decisions aren't anchored to real ownership boundaries, so whether you can legally stop is a guess.

Light & planning siloed

Photography and weather context is missing, and discovery, planning, and navigation never connect into one real trip.

The Insight
Every recommendation should trace to a real fact — a sun vector, a public-land polygon, a road track — not a crowdsourced pin.
Five+ years of research from the road — campfires, pull-offs, dead-end forest roads

How it actually started

The first build was a band-aid.

Ideas came through a messy Procreate session, then a vague Lovable prompt produced a haphazard first generation — impressive and fast for so little direction, but the look just wasn't it. Working backward from an inconsistent UI was only a band-aid while I proved out early ideas.

Where It Broke

Three ways it broke.

AI collapses the cost of building — it doesn't collapse the cost of being wrong. The hardest failures looked correct from every angle.

Hallucinations

The model invented table names and function signatures. The data lied too — OSM contributors disagree on surface tags, PAD-US polygons shift year to year. Surfacing the raw data early caught most of it.

Context ceiling

Photo Scout grew into a 2,606-line monolith the agent could no longer reason about. A refactor — state out of the page, math into a FastAPI service, UI into 500-line panels — cut it to 218 lines. A discipline win, not an AI win.

Emergent bugs

The worst bugs were where every local piece was correct: an RLS policy recursing through trip collaborators, a paved campground labeled "4WD required" because OSM never tagged its interior path. Nothing the model saw was wrong.

Under the Hood

Three structures underneath.

Each pillar started as a clean abstraction — a polygon, a graph, a raster — and ended as a messier problem about how the source data lies, and a product call about how honest to be about it.

01 — A Polygon

Public-land ownership

Boundaries from PAD-US, a USGS aggregate of hundreds of sources at uneven accuracy. A "BLM" polygon can hide a private inholding, and camping rules change state to state.

Tradeoff: PAD-US misses by ~30m at edges. Rather than delete edge spots, I flag distance-to-boundary behind a conservative 50m buffer.
02 — A Graph

Roads & accessibility

Every drivable OSM way as PostGIS lines, each carrying a normalized vehicle-access class the planner treats as a hard constraint. A true dead-end on public land is a strong prior for a real dispersed site.

Tradeoff: The first pass was too generous. Public-land containment plus a half-mile private-road buffer costs a few spots — but failing closed beats failing confidently.
03 — A Raster

Terrain & light

USGS 3DEP tiles as chunked PostGIS rasters. Sun position from first principles, line-of-sight raycast and corrected for curvature — every render shows the actual face the sun will light.

Tradeoff: The first version was mathematically clever and photographically wrong. It had to be scrapped. Moving fast doesn't always equal success.
My Solutions

Pine + Paper.

After the first prototype I stopped building features for two weeks and built the design system. It couldn't look like default SaaS or outdoor-retail marketing — the agent could build either, but couldn't tell me which the data deserved. The north star became The Field Notebook: a warm paper canvas, one ink hand, a primary pine accent.

Specific over evocative — "38 mi of graded gravel, last 4 mi rutted," not "rough road."
Color is never the only signal — a drivability tier is also a glyph and a label, legible in bright sun on a vehicle screen.
Restraint as a contract — the visible promise of the data underneath, not minimalism for its own sake.
Code → Figma via MCP — mirrored live React components into a structured library so design and code stay in lockstep.

What I'd Do Differently

What I'd front-load next time.

01

LLM & agent efficiency

Set up your .md files, use skills, and wire a backlog tool like Notion early — efficiencies I only reached after months of prompting into context-less sessions.

02

Local first

I had a local DB and didn't use it. Prototyping 250k+ rows of road and polygon data against a cloud database will destroy it — test smaller slices locally to avoid huge migrations.

03

Resist side features

Agentic tooling makes almost anything buildable fast, which quietly turns scope into the hard problem. Constraint is a skill.

What's Next

Depth over surface area.

The foundation is live, but behind a waitlist. After the early scope spread, the goal now is making the existing experiences feel polished before adding new ones.

01

User preferences

Tailor the explore content to vehicle type, hiking ability, and other real constraints.

02

Chat cost & quality

Improve the LLM chat feature — per-message cost is a real factor for a personal build.

03

Photo Scout calibration

Field-validate against real sunrises and sunsets, and add a user calibration control for edge cases.

04

Points of interest

Score trails, viewpoints, water, dark-sky and historic sites — and feed them into the planner like campsites already are.

05

Offline-first

Cached tiles, cached spots, and cached itineraries for travel where cell service doesn't reach.

06

Collapse the flow

Discovery, itinerary, and Photo Scout should feel like a single decision, not three separate tabs.
